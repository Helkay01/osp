Project Spotlight: React — How a Facebook Side Project Became the Web's Default UI Library

5 min read
← Back to all posts

React was created by Facebook software engineer Jordan Walke and first used internally on Facebook's newsfeed before being open-sourced in 2013. Its central idea — breaking a user interface into reusable components, each managing its own state — was a significant departure from the templating-heavy approaches common in web development at the time.

React's use of a virtual DOM, an in-memory representation of the UI that gets diffed against the real DOM to calculate minimal updates, was a key part of what made it fast enough for complex, frequently updating interfaces.

The project weathered a licensing controversy of its own: React's original open source license included a patent clause that some companies considered legally risky, prompting Facebook to relicense it under the standard MIT license in 2017 after public pressure from parts of the developer community.

React remains maintained primarily by Meta but accepts contributions from a wide external community, and its component model has directly influenced the design of competing frameworks including Vue and, more recently, Solid and Svelte.
