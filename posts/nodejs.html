Project Spotlight: Node.js — Bringing JavaScript Out of the Browser

5 min read
← Back to all posts

Node.js was created by Ryan Dahl and first released in 2009. It combined Google Chrome's V8 JavaScript engine with an event-driven, non-blocking I/O model, letting developers write server-side code in JavaScript — a language that, until then, had run almost exclusively in browsers.

That non-blocking design was well suited to applications handling many simultaneous connections, such as real-time chat systems and APIs, and it meant front-end and back-end code could increasingly share the same language and, in some cases, the same libraries.

The project went through a significant governance moment in 2014 when a group of contributors forked it as io.js over concerns about the pace of development and decision-making under its then-corporate steward, Joyent. The fork was resolved in 2015 when the two projects merged back together under the neutral governance of the newly formed Node.js Foundation, which later merged into the OpenJS Foundation.

Node.js is released under an MIT-style license and remains foundational to the modern JavaScript ecosystem, both directly and through npm, the package registry that grew alongside it.
